Overview

This document details the properties used by the Server-Side connection for the viewing data integration.

Server-Side Properties


Integration Properties Description
accountId Brightcove Account ID.
CustomerId Value for the unique viewer ID. (e.g., Eloqua viewer ID, Hubspot viewer ID, custom viewer ID, etc.)
videoId Brightcove Video ID.
videoName Brightcove Video Name.
VideoDuration

Total length of the video in seconds.

Note: This may not always be set for live events.
player Brightcove Player ID.
ViewDate Date when viewing activity occurred (YYYY-MM-DD).
MaxPercentagewatched
  • For 'onDemand' views, the highest completion percent for this video by a viewer on a particular day.
  • For 'live' views, the completion percent of a viewer's longest viewed section of the live event.
    • Note: This value may be 0 if the live event has no set video duration. (e.g., there is no VOD asset for the live event).
    		•
    TotalWatchDuration
  • For 'onDemand' views, the user's total watch time for this video is in Seconds on this day.
  • For 'live' views, the user's total watch time of all viewed sections of the live event in Seconds.
    		•
    WatchCount
  • For 'onDemand' views, the number of times a user started this video on this day.
  • For 'live' views, the number of sections of the live event a user has seen.
    		•
    utmSource User-provided UTM source (e.g., source of traffic such as YouTube, Facebook, and Google).
    utmMedium User-provided UTM medium (e.g., marketing medium such as CPC or social media).
    utmCampaign User-provided UTM campaign (e.g., campaign name driving traffic).
    utmTerm User-provided UTM term (e.g., keywords used to search that particular title/content).
    utmContent User-provided UTM content.
    contentOwner

    User-provided content owner for video.

    Note: Use insights_content_owner as a reference when creating custom video fields.*
    customField1

    User-provided custom field for video.

    Note: Use insights_marketing_custom1 as a reference when creating custom video fields.*
    customField2

    User-provided custom field for video.

    Note: Use insights_marketing_custom2 as a reference when creating custom video fields.*
    customField3

    User-provided custom field for video.

    Note: Use insights_marketing_custom3 as a reference when creating custom video fields.*
    presentation The video presentation type can be 'live' or 'onDemand'.
    sections
  • For 'onDemand' views, this is null.
  • This is a list of start/end timestamp pairs for' live' views. Each pair denotes a section of a live event that the viewer has seen. The list is limited to the top 10 sections by length, ordered by the start timestamp.
    		•
    If a user replays or rewinds the video, the viewing time is added to the total watch duration. However, it is not recorded as a new watch count.
    Note: If the custom object in your MAP instance does not have some fields, please update your integration in Video Cloud to add the missing fields.
    *For additional info on creating custom fields in Video Cloud, see here.
